Build a webpage with an AI agent — no code

22 July 2026

Here's the promise: in about twenty minutes you'll have a real webpage open in your browser, and you won't write a single line of code. You'll make it the way a lot of people build things now — by describing what you want to an AI agent and letting it do the work, then telling it what to change.

The agent we'll use is Claude Code. If you watched the agent loop animation, you already know what it is underneath: a model that thinks, uses tools, and loops until the job's done. Claude Code comes as a friendly desktop app — a chat window where you type what you want and it does the work in a folder on your computer. Other tools do the same thing (OpenAI's Codex, Cursor, and a few more); I'm using Claude Code because it's what I use and the steps are concrete. The ideas carry over to any of them.

What you'll end up with

Something like this — a small personal page, built entirely by asking:

file:///my-webpage/index.html
Hi, I'm Alex.
I like building small useful things and learning out loud.
ReadingSci-fi and design books
MakingTiny web experiments
WalkingLong ones, no headphones
Say hi: alex@example.com

Before you start

You need a computer (Mac, Windows, or Linux), about twenty minutes, and a paid Claude plan (Pro, Max, or Team) — the app needs one, and there's no free tier for it, so it's fair to know that up front before you get invested. What you don't need is any coding experience, or the terminal.

Build it with the desktop app

This is the route I'd point anyone to first. No terminal, no typing commands — just an app and a chat box. (Prefer the command line? That's the next section.)

1. Get the Claude Code app

Download the Claude Code desktop app from code.claude.com and install it like any other program — open the installer, click through it. Then open the app and sign in with your Claude account. That's the whole setup: no terminal, nothing to type.

2. Point it at a new folder

Make an empty folder somewhere you'll find it again — say a new folder called my-webpage on your Desktop. In the app, open the Code tab, choose Local, and pick that folder. You've just told the agent where to work.

3. Ask for what you want

In the chat box, type what you'd like in plain English:

You typeBuild a simple one-page personal website for me — a friendly intro, three things I'm into, and a way to reach me. Make it clean and modern, as a single HTML file I can open in my browser.
Claude Code
● Creating index.html…
✓ Done — open index.html to see it.

Send it and watch. This is the loop from the animation, happening for real: it reads your request, decides to create a file, writes the page, and checks that it's there. Decide, act, observe — the whole machine, working for you.

4. Open it, then change your mind

The app creates a file called index.html in your folder. Double-click it and it opens in your browser. That's your webpage — you made it.

Don't love it? Just say so, in the same plain English: "make the heading bigger and use warmer colors," or "add a section about my dog." It revises and you refresh. That back-and-forth is the work now — the steering, not the typing.

5. Make it look genuinely good (optional)

Out of the box the page will look fine. To give it real design polish, you can add a skill — a bundle of expert instructions that makes the agent better at one kind of task. In the app, click + → Plugins and add the free frontend-design skill, then ask again — "rebuild the page using the frontend-design skill" — and the result comes back with a real point of view instead of a default look. Skills are a whole topic of their own; for now it's enough to know they exist and they help.

Build it from the command line

Rather type commands than click around an app? Same result, same plain-English prompt — it's just a little more setup. Skip this entirely if the app worked for you.

1. Open a terminal. That's the Terminal app on a Mac, or Windows Terminal (or PowerShell) on Windows. If you already use an editor like VS Code, its built-in terminal works too — Terminal → New Terminal.

2. Install Claude Code, once. It needs Node.js first (a free download from nodejs.org); then paste:

npm install -g @anthropic-ai/claude-code

3. Make a folder, move into it, and start the agent. The first run signs you in through your browser:

mkdir my-webpage cd my-webpage claude

4. Type the same request in plain English at the prompt. It builds index.html in that folder — open it by double-clicking, same as before.

5. To add the design skill, type these inside Claude Code:

/plugin marketplace add anthropics/claude-code /plugin install frontend-design@claude-code-plugins

Then ask it to redo the page with the skill — type "rebuild the page using the frontend-design skill" at the prompt, and it comes back with real design polish.

Commands change over time — the current official steps are at Anthropic's Claude Code docs.

A few real cautions: agents get things wrong. They'll sometimes produce a broken layout or misread what you meant — that's normal, and correcting it is part of the process, not a sign you did something wrong. Two habits worth keeping: never use something an agent made without looking at it yourself, especially if it touches anything that matters; and remember each run uses up part of your plan, so if it ever seems stuck in a loop, stop it. And no, you didn't cheat. Describing a goal clearly, judging the result, and steering the fixes is a real skill — arguably the one that matters most now.

That's the easy way, and it's also just the way. Curious what the agent is really doing when it "creates a file"? That's a post of its own, coming later. For now: go build something, and change the colors until you like them.
