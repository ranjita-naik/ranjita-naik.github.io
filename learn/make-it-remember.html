Make it remember

23 July 2026

Yesterday you built a webpage by describing it to an agent. It felt a little like magic. Then you came back today, asked for one small change, and the magic curdled: the agent had no idea what you were talking about. The project, your preferences, even your name — gone. You were talking to a stranger who happened to have your files open.

This is the first real wall people hit, and it's worth understanding rather than just working around. So let's do the pain first, then the fix — which is genuinely a five-minute thing once you see why it works.

Tuesday, and it's forgotten everything

Here's Monday. You and the agent build the page together, and by the end it knows the whole shape of the thing.

You · MondayBuild my personal site as one HTML file — British spelling, no emojis, keep it clean.
Claude Code · Monday
● Built index.html — one file, British spelling, no emojis.
✓ Looks good. Anything to change?

Now here's Tuesday. Same project, same folder, one day later.

You · TuesdayMake the header bigger, like we said.
Claude Code · Tuesday
Which header? I don't have any record of a previous conversation or design decisions. Could you tell me about the project?

That blank stare is the thing. And it isn't being difficult; it's doing exactly what it was built to do. To see why, we have to look at where an agent's memory actually lives.

Why doesn't it just… remember?

If you read the agent loop, you saw the one idea that explains this: every new session starts from zero, and the conversation itself is the only thing the model can remember. Everything the agent "knows" in the moment — your goal, the files it looked at, what it decided — lives in a single scratchpad called the context window, and that scratchpad is what makes it seem to remember while you're talking. (Computer scientists have a word for "keeps nothing from last time": stateless. The word matters less than the consequence.)

But when you close the session, that scratchpad is wiped. The model itself remembers nothing from last time — anything that should carry over has to be written down somewhere it will read again. So each new session starts fresh, and unless your context is waiting somewhere it can reload, you end up re-explaining who you are and how you work. That costs you a couple of minutes, and it costs the agent a chunk of its scratchpad — space measured in tokens, the small pieces of text it reads — before any real work starts.

The fix turns out to be almost comically small — one text file, called CLAUDE.md, which we'll open up in a minute. But it lands harder if you feel the problem first. The fix: a file called CLAUDE.md

The solution is surprisingly ordinary. It's just a file called CLAUDE.md that sits in your project folder — a plain text file the agent reads before it starts work, the way you might leave a short handbook on the desk for someone starting a new job. CLAUDE.md is exactly that handbook for your agent: it reads the file automatically at the start of every session, before it does anything else. You write it once; from then on it's re-read at the start of every session automatically — the cost of one small file, not a fresh explanation each time.

That's the entire trick. Not a database, not a setting buried in a menu — a file, in your folder, that the agent reads first, every time.

This idea isn't unique to Claude Code. Most serious agent tools have some version of it — a memory or rules file the agent reads on startup (you'll see the name AGENTS.md elsewhere). So you're learning a habit that travels, not a single vendor's quirk; we just use CLAUDE.md here because it's concrete.

What one actually looks like

Less than you'd think. Here's a real, complete CLAUDE.md for the little website project — five short sections in plain English, with nothing to learn beyond the ## marks that label each one:

CLAUDE.md
# CLAUDE.md — the notes my agent reads first, every session ## The project ← what we're building A one-page personal site, kept in one HTML file. ## House rules ← your always / never list - British spelling ("colour", "organise"). - No emojis, anywhere. - Keep everything in the single index.html file. ## About me ← context it shouldn't have to ask for I'm Ranjita. Plain, friendly tone. Clarity over cleverness.

A good CLAUDE.md is mostly four kinds of thing: what the project is, a couple of firm "always/never" rules, your plain preferences, and any command or fact you find yourself repeating. Keep it short and specific — a tight page beats a sprawling one, for a reason we'll get to.

And you don't have to write it cold. The easiest start is to ask the agent to draft its own: "Look at this project and write a CLAUDE.md capturing what it is and how I've asked you to work." Read what it produces, fix the two lines you disagree with, and you're done.

Tuesday, replayed

With that file in the folder, run Tuesday again. Same request, into a fresh session — but now the agent reads the handbook before it answers.

You · Tuesday (with CLAUDE.md)Make the header bigger, like we said.
Claude Code · Tuesday
● Read CLAUDE.md — personal site, one HTML file, British spelling, no emojis.
✓ Made the header larger in index.html, kept everything in the one file. Want it larger still?

No re-introduction, no interrogation. It picked up where you left off because the context it needed was waiting for it in the file.

A few real cautions

Two caveats worth keeping. CLAUDE.md holds the rules you wrote down, not a memory of yesterday's actual chat — a handbook, not a diary. And because it's re-read on every session, a bloated file quietly taxes every run, which is the whole reason to keep it short. Neither is a reason to skip it; that's just how to use it well.

CLAUDE.md also isn't the only memory at work. Claude Code now keeps a few notes on its own between sessions, without being asked — a light automatic memory. That's handy, but you don't choose what it saves; CLAUDE.md is the part you set deliberately, so the rules you care about are followed every time rather than left to the agent's guess.

The one thing it still can't do

So now your agent remembers who you are and how you work. That's a real step forward — the kind of habit that pays off every time you start a new session. But before long you'll run into a different kind of problem.

Remembering isn't the same as reaching.

Your agent still can't open your code on GitHub, check your calendar, or read a live page on the web. Its memory got better; its arms are still too short.

By now, you may have noticed a pattern.
Yesterday's frustration was "it forgot everything," and the answer was CLAUDE.md.
The next frustration is just as plain: "Why can't it open GitHub?"
It turns out to have a surprisingly simple answer too.

Try it on something real before then: open the project you built yesterday, ask the agent to draft a CLAUDE.md for it, and tweak a line or two. Three minutes now saves you every re-introduction after.
