The agent loop, animated

20 July 2026

Every AI agent — no matter the framework, no matter the vendor — is the same small machine: a model that thinks, acts through tools, observes what happened, and loops until the goal is done. Below, a real (simulated) agent answers the question "How much did we spend on software subscriptions last quarter?" Step through it and watch the machine run. Then flip on "inject a failure" to see the part that makes it an agent and not a script.

What you just watched

Three things are worth noticing. First, the model never touches the world directly — it only emits structured requests ("call read_file with this path"), and ordinary code executes them. Second, the context window is the agent's entire memory: every thought, call, and result gets appended, which is why long tasks get expensive and why "context engineering" is a real discipline. Third — flip the failure switch if you haven't — the loop is the intelligence. A script would have crashed on the error; the agent reads the error as just another observation and routes around it. That single property is most of what separates agents from automation.
